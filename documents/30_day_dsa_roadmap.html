Dynamic programming patterns

DP is just "memoized recursion." Learn the 5 patterns and you can crack 90% of DP questions. Never memorize solutions — memorize the decision framework.

1D DP
Days 19–20
  • E
    Climbing Stairs
    Pattern: dp[i] = dp[i-1]+dp[i-2]
  • E
    House Robber
    Pattern: skip or take
  • M
    Longest Increasing Subsequence
    Pattern: patience sort / dp
  • M
    Word Break
    Pattern: dp[i] = any valid split
  • M
    Coin Change
    Pattern: unbounded knapsack
2D DP
Days 21–22
  • M
    Unique Paths
    Pattern: grid DP (down+right)
  • M
    Longest Common Subsequence
    Pattern: match or skip dp[i][j]
  • M
    Edit Distance
    Pattern: 3-way choice at each cell
  • M
    0/1 Knapsack
    Pattern: include/exclude item
  • H
    Burst Balloons
    Pattern: interval DP (last burst)
Backtracking
Days 23–24
  • M
    Subsets
    Pattern: include/exclude DFS
  • M
    Combination Sum
    Pattern: reuse + pruning
  • M
    Permutations
    Pattern: swap-based backtrack
  • M
    Palindrome Partitioning
    Pattern: DFS + palindrome check
  • H
    N-Queens
    Pattern: constraint propagation
Phase tip

DP decision tree: Can I break this into overlapping subproblems? Yes → DP. Does the subproblem have one variable (index)? 1D. Two? 2D. Draw the recurrence before writing code.
